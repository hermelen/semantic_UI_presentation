1) Button :
- Personnalisation du bouton plus enrichie.
- Plus d’animation.
- Plus de choix de couleurs.
2) Input :
- Un champ de saisie d'icône peut montrer qu'il charge actuellement des données (loading).
- Peut montrer que les données contiennent des erreurs (error).
- Une entrée peut être formatée avec une icone (icon), un label (labeled).
- Peut alerter l'utilisateur d'une action qu'il peut effectuer (action).
3) Label :
- Une étiquette peut être formatée pour mettre en valeur une img (image).
- Peut pointer sur du contenu à côté d'elle.
- Elle peut apparaître comme une étiquette/ruban (ribbon) et se mettre sur les éléments (tag).
- S'attache à un segment du contenu (attached).
- Peut flotter au dessus d'un autre élément (floating).
- Peut être circulaire (circular) et coloré.
4) Image :
- Une image peut être formatée pour lier un autre contenu (Image Link).
- (Une image peut être formatée pour apparaître en ligne avec du texte en tant qu'avatar).
- L'image prend la taille de son container (Fluid).
- Elle peut apparaître directement avec les bords arrondies ou circulaire (rounded/circular).
- Elle peut se mettre à droite ou à gauche d'un autre contenu (Floated).
Quelques éléments qui sont en plus :
1) Flag : On peut rajouter les drapeaux à côté du nom des pays.
2) Icon : Des icônes sont directement intégrés dans "Semantic UI".
3) Loader : Plusieurs chargements de tailles différentes sur tout contenu voulu avec le positionnement que l'on souhaite.
4) Rail : Affiche le contenu d'accompagnement facultatif en dehors de la fenêtre principale de notre site Web.
5) Reveal : Animation en plus qui sont sur les img. (Fade)
- Peut contenir du contenu visible et caché avant l'interaction de l'utilisateur (visible content/hidden content).
- Peut montrer son contenu sans délai (instant).
6) Segment : est utilisé pour créer un regroupement de contenu associé.
7) Step : Montre le statut d'achèvement d'une activité dans une série d'activités (ex : finalisation d'un achat, montre les étapes par étape).
II) Collections.
Les différences entre Semantic UI et Bootstrap :
1) Grid :
- Syntaxe beaucoup plus simple.
- Grille 16.
- Pour le responsive : beaucoup plus intuitif (mobile à la place de sm-md...)
- Certains types de grilles (divisions ou les cellules), nécessitent des wrappers de lignes pour appliquer le format correctement (Special Grids).
2) Menu :
- Pagination : Spécialement formaté pour présenter des liens vers des pages de contenu.
- Dropdown Item : Un élément peut contenie un menu imbriqué dans une liste déroulante.
- Popup Menu : Peut afficher un menu volumineux ou du contenu supplémentaire à l'aide d'une fenêtre contextuelle.
- Search : Le menu peut contenir un search input.
- Sub Menu : Un élément de menu peut contenir un autre menu imbriqué à l’intérieur qui agit comme un sous-menu groupé.
- Hover : Peut être survolé.
- Stackable : Peut s'empiler aux résolutions mobiles.
- Colored / Icons : spécifié les couleurs que l'on veut sur le menu, faire un menu juste avec des icones.
- Labeled Icon : Peut avoir des icônes étiquettées.
- Attached : Peut être joint à d'autres segments de contenu.
3) Message :
- Icon Message : Peut contenir des icônes dans le contenu.
- Dismissable Block : L'utilisateur peut choisir de cacher le message.
- Colored : Les messages peuvent avoir des couleurs différentes.
4) Table : (elles ne sont pas présentes dans Bootstrap).
Quelques exemples :
- Table classique en ajoutant quelques éléments comme des icônes et des messages de couleurs (alertes).
- Une table peut être formatée pour mettre en évidence une première colonne qui définit un contenu de lignes.
- Elle peut être formatée pour afficher des donnée structurées complexes.
- States : Une cellule ou une ligne peut permettre à un utilisateur de savoir si une valeur est bonne ou mauvaise.
- Cell (error) : Une cellule ou une ligne peut attirer l'attention sur une erreur ou une valeur négative.
- Warning : Une cellule ou une ligne peut avertir un utilisateur.
- Active/Disabled : Peut être active/désactivée ou sélectionnée par un utilisateur.
Quelques éléments qui sont en plus :
1) Breadcumb : Un fil d'Ariane est utilis pour montrer la hiérarchie entre le contenu.
2) Advertisement : Affiche le contenu promotionnel d'une tierce partie (publicité sur les sites web).
3) Card : Affiche le contenu du site d'une manière similaire à une carte à jouer (une carte personnalisée, flex où on peut ajouter les options que l'on souhaite).
4) Comment : Affiche les commentaires des utilisateurs sur le contenu du site.
5) Feed : Un flux présente l'activité de l'utilisateur par ordre chronologique.
6) Statistic : Met l'accent sur la valeur actuelle d'un attribut.
III) Modules.
Ceux-ci ne sont pas présents dans le Bootstrap :
1) Accordion : Permet aux utilisateurs de basculer l'affichage des sections de contenu.
2) Checkbox.
3) Dimmer : Un gradateur masque les distractions pour attirer l'attention sur un contenu particulier.
4) Dropdown : On peut les personnaliser et les animer comme on le souhaite.
5) Embed : Affiche le contenu d'autres sites (ex : YouTube ou Google Maps).
6) Popup :On peut les ajouter sur des articles, boutons, img...
7) Progress : Montre la progression d'une tâche. Elle peut avoir plusieurs styles, couleurs. Ca peut être aussi de simple bordures qui montrent le chargement.
8) Rating : Indique l'intérêt des utilisateurs pour le contenu (des étoiles/coeurs avec de différentes tailles).
9) Shape : Une forme est un objet tridimensionnel affiché sur un plan en deux dimensions (switch les images en donnant les directions voulues). On peut également l'utiliser sur les textes.
10) Sticky : Le contenu collant se fixe dans la fenêtre du navigateur lorque le contenu défile.
IV) Behaviors.
1) API : Permet aux éléments de déclencher des actions sur un serveur.
2) Form Validation : Un comportement de validation de formulaire vérifie les données par rapport à un ensemble de critères avant de le transmettre au serveur.
3) Visibility : La visibilité fournit un ensemble de rappels lorsqu'un contenu apparaît dans la fenêtre.
